Phase 1

Organise Your Files

Before anything else, get your folder clean and ready on your computer.

1
Create the project folder

On your Desktop or Documents, create a new folder. Name it exactly:

santhosh-portfolio
Tip
No spaces in the folder name. This name will be part of your website URL on Vercel.
2
Download all files from Claude and save them

Download every file that was generated and place them like this:

santhosh-portfolio/ │ ├── index.html ← Main portfolio website ├── dashboard-sv2024.html ← Admin panel ├── DEPLOY_GUIDE.html ← This guide │ └── assets/ └── photo.jpg ← YOUR photo (add this)
Important
The assets/ folder must be inside santhosh-portfolio/. Do not rename any of the HTML files — their exact filenames are used in links.
3
Add your profile photo

Save your profile photo as photo.jpg inside the assets/ folder.

Then open index.html in Notepad or VS Code. Press Ctrl+F and search for:

TO ADD PHOTO

Find these 2 lines and delete them:

<div class="acard-mono">SV</div> <p class="acard-hint">Add your photo</p>

Paste this in their place:

<img src="assets/photo.jpg" alt="Santhosh V" style="position:absolute;inset:0;width:100%; height:100%;object-fit:cover;object-position:top">

Save the file. Photo is now connected ✓

Photo Tips
Best size: at least 600×750px portrait. JPG format. Make sure your face is centred. If you don't have a photo yet, skip this — the monogram will show instead.
4
Update your social media links

Open index.html in a text editor. Press Ctrl+F and search for:

href="#" class="soc"

You'll find 4 of them. Replace each # with your real profile links:

Instagram: https://instagram.com/YOUR_USERNAME YouTube: https://youtube.com/@YOUR_CHANNEL LinkedIn: https://linkedin.com/in/YOUR_PROFILE WhatsApp: Already set to +91 9994723048 ✓

Save the file.

Phase 2

Upload to GitHub

GitHub stores your files in the cloud so Vercel can deploy them. It's free and takes 5 minutes.

1
Create a free GitHub account

Go to github.com and click Sign Up.

Use any email — I suggest the same Gmail you use for EmailJS: santhoshwe2007@gmail.com

Complete the signup, verify your email. Done.

Already have GitHub?
Just log in and skip to the next step.
2
Create a new repository

After logging in, click the + icon (top right) → New repository

Fill in the form exactly like this:

FieldWhat to enter
Repository namesanthosh-portfolio
DescriptionMy portfolio website (optional)
Visibility✅ Public (required for free Vercel)
Add READMELeave unchecked

Click the green Create repository button.

3
Upload your files to GitHub

On your new empty repository page, look for the link that says "uploading an existing file" — click it.

You'll see a drag-and-drop area. Now:

1. Open your santhosh-portfolio folder on your computer

2. Select ALL files: index.html, dashboard-sv2024.html, DEPLOY_GUIDE.html

3. Also drag the entire assets folder into the upload area

4. Wait for all files to upload (green checkmarks appear)

Important
GitHub doesn't preserve folder structure when you drag files directly. You must also drag the assets folder separately or GitHub will ignore it. Alternatively, drag the entire santhosh-portfolio folder into the upload area.

5. Scroll down to the "Commit changes" section

6. Leave the commit message as default or type: Initial upload

7. Click the green Commit changes button

Your files are now on GitHub. ✓

Phase 3

Deploy on Vercel — Make It Live

Vercel reads your GitHub files and gives you a live URL instantly. Free forever for personal portfolios.

1
Create a Vercel account

Go to vercel.com → click Sign Up

Choose Continue with GitHub — this connects them automatically.

Authorise Vercel when GitHub asks for permission. You'll land on the Vercel dashboard.

2
Import your repository

On the Vercel dashboard, click Add New… → Project

You'll see a list of your GitHub repositories. Find santhosh-portfolio and click Import.

Don't see your repo?
Click "Adjust GitHub App Permissions" and grant Vercel access to your santhosh-portfolio repository.
3
Configure and deploy

On the configuration screen, you don't need to change anything. Leave all settings as default:

SettingValue
Framework PresetOther (leave as is)
Root Directory/ (leave as is)
Build CommandLeave empty
Output DirectoryLeave empty

Click the big Deploy button. Vercel builds in 10–20 seconds.

You'll see a congratulations screen with your live URL. It looks like:

https://santhosh-portfolio.vercel.app
Your site is live!
Click "Visit" to open your portfolio. Share this link with anyone — it's your live portfolio URL.
4
Rename your URL (optional but recommended)

In the Vercel dashboard, open your project → Settings → Domains

You can change the free subdomain. For example:

santhoshv.vercel.app santhoshdesigns.vercel.app santhoshcreates.vercel.app

Type your preferred name, click Add. Vercel checks if it's available and sets it up instantly.

Custom domain like santhoshv.in?
Buy from GoDaddy or Bigrock (~₹600/year). Then go to Vercel → Settings → Domains → Add your domain → Follow the DNS instructions shown. Takes 5–10 minutes to go live.
5
How to update your site in the future

Vercel auto-deploys whenever you update your GitHub files. To update:

1. Go to your GitHub repo → click on the file you want to edit (e.g. index.html)

2. Click the pencil icon (Edit this file)

3. Make your changes → scroll down → click Commit changes

4. Vercel detects the change and deploys automatically in ~20 seconds

Tip
You can also drag new files (like portfolio images) directly to your GitHub repo by going to the repo page → Add file → Upload files.

5
How to use the Delivery Tracker for each client

When a client places an order:

1. You get an email from EmailJS with their order details

2. Open admin panel → Orders page

3. Find their order → click the Track button (green)

4. Their details auto-fill in the Delivery Tracker form

5. Set the estimated delivery date → add a note → click Save Tracker

As work progresses:

6. Go to the tracker card → click Next Stage → to advance

7. Click Edit to update the note at any stage (e.g. "Design done, sending for review")

8. When done → advance to Stage 5: Delivered ✓

6
How to generate and download a Review Poster

When a client leaves a good review, turn it into a shareable image in 3 clicks:

1. Admin panel → Reviews page

2. Find the review → click the Poster button

3. Or: scroll to the top of the Reviews page → use the Review Poster Generator

4. Select review from dropdown → choose Dark / Lime / Minimal style

5. Edit the tagline text if needed

6. Click Download PNG — saves a 1080×1080px image

7. Post it on Instagram, WhatsApp status, or Stories

What the poster includes
Star rating, full review quote, client name & role, your SS logo, and your tagline — all branded with your aesthetic.
7
How to add real portfolio images

Right now your portfolio shows SVG design compositions. To show your actual work:

1. Go to imgbb.com → click Start Uploading

2. Upload your thumbnail/poster/logo image

3. After upload, click on the image → copy the Direct Link URL (ends in .jpg or .png)

4. Admin panel → Portfolio → find the item → click Edit

5. Paste the imgbb URL in the Image URL field → Save

Your real image now shows on the portfolio site instantly.

Help

Troubleshooting Common Issues

ProblemFix
Name "SANTHO" appears cut offThe fix was already applied in the latest index.html. Re-download and re-upload.
Email not arriving in Gmail1. Check EmailJS dashboard for error logs. 2. Verify all 4 keys are correct in index.html with no extra spaces. 3. Check Gmail spam folder. 4. Make sure Gmail service is connected in EmailJS.
Admin panel login not workingPassword is case-sensitive. Default is santhosh2024. If you changed it, open dashboard-sv2024.html on GitHub and check the AP variable.
Portfolio shows SVG designs, not my imagesAdd real images via imgbb.com and paste the URL in Admin → Portfolio → Edit item → Image URL field.
Reviews not saving between visitsReviews use localStorage — they're tied to the browser on your Vercel domain. Works fine once hosted. On local file:// URLs it may not persist.
Photo not showing in About sectionMake sure file is exactly photo.jpg (lowercase) inside the assets/ folder, and you updated the img tag in index.html.
Vercel shows 404 errorMake sure index.html is in the root of your repo (not inside a subfolder). Go to GitHub repo and confirm index.html is visible at the top level.
Fonts look different / not loadingFonts load from Google CDN and need an internet connection. On slow connections they may fall back to system fonts temporarily.
Review poster download not workingSafari sometimes blocks canvas downloads. Use Chrome or Firefox to download posters.
